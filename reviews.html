Örnek 01 — Gereksiz Koşul Yapısı (JavaScript)

Uyarı 
function aktifMi(user) {
  if (user.isActive == true) {
    return true;
  } else {
    return false;
  }
}

İnceleme Notu

Koşul yapısı gereksiz yere uzatılmış. Boolean değer doğrudan döndürülebilir. Ayrıca `==` yerine daha güvenli karşılaştırmalar tercih edilmelidir. 

function aktifMi(user) {
  return user.isActive;
}

Örnek 02 — Daha Temiz Toplama İşlemi (Python)

Bilgi 
total = 0
for item in products:
    total += item["price"]

print(total)

İnceleme Notu

Kod çalışıyor ancak Python’un güçlü built-in özellikleri kullanılmamış. Daha okunabilir ve kısa bir çözüm için `sum()` tercih edilebilir. 

total = sum(item["price"] for item in products)
print(total)

Örnek 03 — Null Kontrolü Eksikliği (Java)

Kritik 
public String getName(User user) {
    return user.getName();
}

İnceleme Notu

`user` nesnesi null gelirse uygulama hata verebilir. Kullanımdan önce null kontrolü yapılmalıdır. 

public String getName(User user) {
    if (user == null) {
        return "Bilinmiyor";
    }
    return user.getName();
}

Örnek 04 — Bellek Yönetimi Riski (C++)

Kritik 
int* number = new int(10);
std::cout << *number << std::endl;

İnceleme Notu

Ham pointer kullanımı bellek sızıntısına yol açabilir. Modern C++ yaklaşımında akıllı pointer kullanılması daha güvenlidir. 

#include <memory>

auto number = std::make_unique<int>(10);
std::cout << *number << std::endl;

Örnek 05 — Zayıf Karşılaştırma (PHP)

Uyarı 
if ($value == 0) {
    echo "Sıfır";
}

İnceleme Notu

Zayıf karşılaştırma beklenmeyen sonuçlara neden olabilir. Daha güvenilir sonuçlar için strict comparison kullanılmalıdır. 

if ($value === 0) {
    echo "Sıfır";
}
